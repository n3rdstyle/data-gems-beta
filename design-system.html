Spacing Systematik: Screens & Modals

Einheitliche Regeln für Padding und Gaps über alle Screens und Modals

1. Screen/Modal Level

Screens (.home, .settings, .profile) und Modals definieren:
  • Padding: var(--spacing-sm) (16px) auf allen Seiten
  • Gap: var(--spacing-screen-internal) (24px) zwischen Components
  • Overflow: hidden am Container
.home, .settings, .profile,
.messages-modal, .data-editor-modal, .tag-add-modal {
  padding: var(--spacing-sm); /* 16px */
  display: flex;
  flex-direction: column;
  gap: var(--spacing-screen-internal); /* 24px */
  overflow: hidden;
}

2. Section/Content Level

Sections innerhalb von Screens/Modals haben:
  • Kein eigenes Padding – Screen/Modal stellt bereits Padding bereit
  • Scrollbare Content-Bereiche haben overflow-y: auto
  • Gaps für interne Abstände zwischen Sub-Sections
.profile__content, .settings__content {
  flex: 1;
  overflow-y: auto;
  overflow-x: hidden;
  display: flex;
  flex-direction: column;
  gap: var(--spacing-screen-internal); /* 24px between sections */
  /* No padding - screen provides padding */
}

3. Component Level

Components (Header, Content-Preferences, etc.) haben:
  • Kein horizontales Padding – wird vom Screen/Modal bereitgestellt
  • Gaps für interne Elemente: var(--spacing-component-internal) (16px)
  • Ausnahmen nutzen negative Margins (z.B. Calibration für full-width)
.content-preferences, .header {
  display: flex;
  flex-direction: column;
  gap: var(--spacing-component-internal); /* 16px */
  /* No padding - screen provides padding */
}

/* Exception: Full-width element */
.home__calibration {
  margin-left: calc(-1 * var(--spacing-sm)); /* -16px */
  margin-right: calc(-1 * var(--spacing-sm)); /* -16px */
}

Apply shadows with CSS variables 

/* Using scale values */
.card {
  box-shadow: var(--shadow-sm);
}

/* Using semantic aliases */
.modal {
  box-shadow: var(--shadow-modal); /* same as --shadow-lg */

.button:hover {

/* Remove shadow */
